Oracle 인덱스 최적화

인덱스는 쿼리 성능을 크게 향상시킬 수 있는 중요한 데이터베이스 객체입니다. 적절한 인덱스 선택과 활용은 쿼리 튜닝의 핵심입니다.

인덱스를 생성해야 하는 경우

  • WHERE 절에 자주 사용되는 컬럼
  • JOIN 조건에 사용되는 컬럼
  • ORDER BY 절에 사용되는 컬럼
  • 선택도(Selectivity)가 높은 컬럼
  • 테이블 크기가 큰 경우

인덱스를 생성하지 말아야 하는 경우

  • 테이블이 작은 경우 (수천 건 이하)
  • 컬럼 값이 자주 변경되는 경우
  • Cardinality가 매우 낮은 컬럼 (비트맵 제외)
  • 테이블의 대부분 행을 조회하는 경우
