BeanPostProcessor(빈 후처리기)

Spring Bean 을 등록하는 시점에 특정 로직을 수행하기 위해 사용하는 hook (LifeCycle 에 간섭할 수 있다.)
Spring Bean 전체에 공통 로직을 수행할 때 사용하면 유용하다

Spring의 BeanPostProcessor 는 스프링 빈의 생성과 초기화를 제어하는 인터페이스입니다. 이 인터페이스를 사용하면 스프링 빈이 생성되고 초기화되는 과정에서 추가적인 작업을 수행할 수 있습니다.

BeanPostProcessor 를 사용하면 스프링 빈의 생성과 초기화 과정에서 추가적인 작업을 수행할 수 있습니다. 예를 들어, 스프링 빈의 필드를 초기화하거나, 빈 객체의 프록시를 생성하는 등의 작업을 수행할 수 있습니다.

BeanPostProcessor 동작 과정

  1. 생성
    • 스프링 Bean 대상이 되는 객체를 생성한다.
  2. 전달
    • 생성된 객체를 Bean 으로 등록하기 전에 빈 후처리기 에 전달한다.
  3. 후 처리 작업
    • 빈 후처리기는 전달된 스프링 Bean 객체를 조작하거나 다른 객체로 바꿔치기 할 수 있다.
  4. 등록
    • 빈 후처리기는 빈을 반환한다.

BeanPostProcessor 인터페이스

Bean 후 처리기를 이용해 Bean 객체를 프록시로 교체하는 것이 가능하다

  • postProcessBeforeInitialization
    • 객체 생성 이후 Bean 이 초기화되기 전에 호출되는 Post Processor
  • postProcessAfterInitialization
    • 객체 생성 이후 Bean 이 초기화된 후에 호출되는 Post Processor

빈 후처리기 적용

빈 후처리기 생성

BeanPostProcessor 인터페이스를 구현한 AToBPostProcessor 에서는 Bean 이 초기화 된후 넘어온 객체가 A 일 경우 B 객체로 새로 생성해 반환한다.

Spring 핵심원리 고급편 - 빈 후처리기

스프링 부트가 등록하는 모든 빈들이 빈 후처리기에 넘어온다. 그래서 어떤 빈을 Proxy Bean 으로 생성할지에 대한 기준 이 필요하다.

BeanPostProcessor 에서 Bean 생성시 ProxyFactory 객체를 생성후 Advisor 를 등록하는 로직을 수행함으로 써 Config 파일에서 ProxyFactory 객체 생성 및 Advisor 를 등록할 필요가 없어졌다. 덕분에 코드가 중복으로 작성되는 문제점 또한 해결 됐다. 다만 모든 Bean 에 대해 BeanPostProcessor 로직이 수행되므로 어떤 Bean 을 Proxy Bean 으로 생성할지에 대한 기준이 필요하다.

  • PackageLogTracePostProcessor 는 넘어온 Bean 의 Package 를 확인 후 Proxy 객체 를 생성한다.
  • Package 이름이 설정된 basePackage 로 시작하면 Proxy 객체를 반환하고
  • 아닐 경우에는 그대로를 다시 반환한다.

BeanPostProcessor 등록

  • logTracePostProcessor
    • 특정 패키지를 기준으로 Proxy 를 생성하는 BeanProcessor 를 Spring Bean 으로 등록 한다.
    • BeanProcessor 를 Bean 으로 등록하면 자동으로 동작한다.
